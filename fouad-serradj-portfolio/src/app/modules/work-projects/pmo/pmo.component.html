Facilities Management Central

FM Central is solution to normalize and standardize the way data is entered and managed. It aims to build an authoritative source of data. Through this central interface, professionals will be able to collect, enter and update data in a coordinated and standardized way. To ensure data uniformity, several layers of data validation are implemented based on Concordia University and Facilities Management business rules and governance.

The FM Central is based on a decoupled architecture. Frontend code is separated from the backend code. The frontend and backend code communicate, but have their own server instances. The backend application serves data via an API in a JSON format, as part of a headless set-up. A decoupled architecture makes it easy to build flexibility and integrate other services that consume such API's like PowerBI. it also allows to make rapid changes, and scale individual services and components independently without disrupting the entire application.
